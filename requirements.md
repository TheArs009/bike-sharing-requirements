# Sistema de préstamo de bicicletas

## Stakeholders
- Estudiantes
- Docentes
- Turistas
- Egresados
- Personal de seguridad
- Directivos

## Requisitos funcionales
- RF1: El sistema debe permitir el registro de los usuarios al sistema
- RF2: sistema debe de informar sobre la disponibiblidad de las bicibletas
- RF3: El sistema debe permitir registrar el prestamo de la bicicleta
- RF4: El sistema debe permitir registrar la devoluvion de las bicicletas
- RF5: El sistema debe de registrar el tiempo de uso de la bicicleta
- RF6: El sistema debe informar los requisitos necesarios para el préstamo de 1 bicicleta

## Requisitos no funcionales
- RNF1: El sistema debe de dar respuesta en menos de 1 segundo
- RNF2: El sistema debe garantizar la seguridad de los datos personales de los usuarios.
- RNF3: El sistema debe de funcionar el 100% del tiempo de atencion al cliente
- RNF4: El sistema debe de ser funcional tanto en dispositivos moviles como en computadores
